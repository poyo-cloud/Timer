# 匂い研究 本実験スケジュール

外部の計測機器で確認した時刻を入力すると、本実験の手順に沿って次の指示時刻を算出する軽量 Web アプリです。

## できること

- パターン A〜H の選択
- 導入手順として「香りA提示 → 評価・reaction time → 香りB提示 → 評価・reaction time → クレペリンテスト 3分」を記録
- クレペリンテストは最初の1回だけ
- 条件 1〜7 の順番管理
- 各条件で「安静 60秒 → 30秒ブロック4回 → 安静 60秒」の指示時刻を表示
- 条件ごとのメモ記録
- 区間時刻つき CSV の書き出し
- 保存済み履歴からの CSV 再書き出し
- 端末内のローカル履歴保存

- 条件1: 安静 60秒、安静 30秒、安静 30秒、安静 30秒、安静 30秒、安静 60秒
- 条件2: 安静 60秒、香りA 30秒、香りA 30秒、香りA 30秒、香りA 30秒、安静 60秒
- 条件3: 安静 60秒、香りA 30秒、安静 30秒、安静 30秒、安静 30秒、安静 60秒
- 条件4: 安静 60秒、香りA 30秒、安静 30秒、香りA 30秒、安静 30秒、安静 60秒
- 条件5: 安静 60秒、香りB 30秒、香りB 30秒、香りB 30秒、香りB 30秒、安静 60秒
- 条件6: 安静 60秒、香りB 30秒、安静 30秒、安静 30秒、安静 30秒、安静 60秒
- 条件7: 安静 60秒、香りB 30秒、安静 30秒、香りB 30秒、安静 30秒、安静 60秒

## パターン

- パターンA: 条件3 → 条件7 → 条件1 → 条件5 → 条件2 → 条件6 → 条件4
- パターンB: 条件6 → 条件2 → 条件4 → 条件7 → 条件1 → 条件3 → 条件5
- パターンC: 条件1 → 条件5 → 条件7 → 条件2 → 条件6 → 条件4 → 条件3
- パターンD: 条件4 → 条件1 → 条件6 → 条件3 → 条件7 → 条件5 → 条件2
- パターンE: 条件7 → 条件4 → 条件2 → 条件6 → 条件3 → 条件1 → 条件5
- パターンF: 条件2 → 条件6 → 条件5 → 条件1 → 条件4 → 条件7 → 条件3
- パターンG: 条件5 → 条件3 → 条件6 → 条件4 → 条件1 → 条件2 → 条件7
- パターンH: 条件6 → 条件7 → 条件3 → 条件2 → 条件5 → 条件4 → 条件1

## 本実験の流れ

1. 質問票（被験者特性、STAI、NEO-FFI、CD-RISC）
2. 香りA提示
3. VAS 6項目、スタンフォード眠気尺度日本語版、reaction time（視覚3回・聴覚3回）
4. 香りB提示
5. VAS 6項目、スタンフォード眠気尺度日本語版、reaction time（視覚3回・聴覚3回）
6. クレペリンテスト 3分
7. パターンに沿って条件 1〜7 を実施
8. 各条件後に VAS 6項目、スタンフォード眠気尺度日本語版、reaction time（視覚3回・聴覚3回）

## 使い方

1. `index.html` をブラウザで開く
2. 参加者IDまたはセッション名を入力する
3. パターン A〜H を選ぶ
4. 導入では、香りA/B提示・評価・reaction time 後にクレペリン開始時刻を入力して `指示を作成`
5. 各条件では、条件開始時刻（最初の安静開始）を入力して `指示を作成`
6. 実施後に必要ならメモを書き、`この条件を記録して次へ`
7. すべて終わったら CSV を書き出す
8. 過去セッションは履歴欄から再度 CSV を書き出せます

## 前提

- アプリ自体ではタイマー計測を行いません
- 入力された開始時刻を基準に、区間開始・終了時刻を計算します
- 保存履歴はこのブラウザの `localStorage` に保存されます
- 出力CSVは本実験用に、導入・条件1〜7を含む形式です
- 出力CSVのファイル名は `Timer_参加者ID.csv` です
- 生データとの突き合わせや平均値計算は別ツールで行う前提です
